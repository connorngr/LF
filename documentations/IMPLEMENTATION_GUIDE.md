# Life Frame - Implementation Guide

## Tech Stack Overview

### Full Stack (Next.js)
- **Framework**: Next.js 16+ (React 19+)
- **Deployment**: Vercel
- **Styling**: TailwindCSS + shadcn/ui (for polished, minimal UI)
- **Database**: Supabase PostgreSQL
- **File Storage**: Cloudflare R2
- **Authentication**: Environment variable-based hardcoded admin credentials
- **API Routes**: Next.js API routes (built-in, no separate backend needed)

### Single Repository Structure
- **Package Manager**: npm/yarn
- **Structure**:
  ```
  lifeframe/
  ├── app/
  │   ├── page.tsx              (homepage)
  │   ├── layout.tsx            (root layout)
  │   ├── api/                  (API routes)
  │   │   ├── images/

  │   │   │   ├── random/route.ts (GET random image)
  ```

## Database Schema (Supabase PostgreSQL)

### Tables

#### `images`
```sql
CREATE TABLE images (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  image_url TEXT NOT NULL,
  folder_name TEXT NOT NULL,
  caption TEXT,
  date_taken DATE NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_folder_name ON images(folder_name);
CREATE INDEX idx_date_taken ON images(date_taken);
```

#### `folders`
```sql
CREATE TABLE folders (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT UNIQUE NOT NULL,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## API Endpoints (Next.js API Routes)

### Public Routes (Read-only)
- `GET /api/images` - Get all images (paginated)
- `GET /api/images/random` - Get random image for the day
- `GET /api/folders` - Get all folders
- `GET /api/images/[id]` - Get single image details

### Admin Routes (Requires auth)
- `POST /api/images/upload` - Upload image to R2 + create DB record
- `POST /api/folders` - Create new folder
- `PATCH /api/images/[id]` - Update image caption
- `DELETE /api/images/[id]` - Delete image

### Authentication
- Simple middleware checking `Authorization: Bearer <ADMIN_TOKEN>` header
- Token generated by comparing `ADMIN_PASSWORD` env var on login
- Token stored in localStorage on frontend
- Middleware applied to protected API routes in `app/api/middleware.ts`

## Environment Variables

### .env.local (Next.js)
```
# Supabase
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key

# Cloudflare R2
CLOUDFLARE_ACCOUNT_ID=your-account-id
CLOUDFLARE_ACCESS_KEY_ID=your-access-key
CLOUDFLARE_SECRET_ACCESS_KEY=your-secret-key
CLOUDFLARE_R2_BUCKET_NAME=lifeframe
CLOUDFLARE_R2_PUBLIC_URL=https://your-r2-bucket.your-domain.com

# Admin Authentication
ADMIN_PASSWORD=your-secure-password

# Environment
NODE_ENV=production
```

**Note**: Variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. Keep sensitive keys (like `SUPABASE_SERVICE_ROLE_KEY`) server-only.

## Daily Random Image Logic

### Algorithm
1. Use `date_taken` to seed a deterministic random function
2. On each request to `/api/images/random`:
   - Get today's date (UTC)
   - Hash the date to get a seed
   - Use seed to pick a random image from the database
   - Same image shown all day, different image next day

### Implementation
```javascript
// Backend utility
function getDailyRandomSeed() {
  const today = new Date().toISOString().split('T')[0];
  return parseInt(today.replace(/-/g, ''), 10);
}

// Deterministic random pick
function getRandomImageForDay(images, seed) {
  const index = seed % images.length;
  return images[index];
}
```

## Cloudflare R2 Integration

### Setup Steps
1. Create R2 bucket in Cloudflare dashboard
2. Generate API token with R2 permissions
3. Use `aws-sdk` or `@aws-sdk/client-s3` to upload files
4. Store public R2 URL in database

### Upload Flow
1. Frontend sends image file to `/api/images/upload` (multipart/form-data)
2. Backend validates admin token
3. Backend uploads to R2 with unique key: `{folder}/{timestamp}-{uuid}.jpg`
4. Backend creates database record with R2 public URL
5. Return image record to frontend

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| Next.js Full Stack | Single framework, single deployment, no FE/BE separation complexity. |
| API Routes | Built-in, no external server needed. Scales with Vercel. |
| Cloudflare R2 | Cheap, reliable, good image quality preservation. |
| Supabase | Managed PostgreSQL + auth ready if needed later. |
| Env var auth | Simplest possible admin auth for MVP. |
| Daily random | Deterministic, creates habit of checking daily. |
| Public read-only | No complexity of user permissions yet. |
| Vercel | Best DX for Next.js, auto-deploy, free tier, built-in serverless. |
